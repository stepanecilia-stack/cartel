# Cartel Boxing App

Веб-приложение для тренера по боксу: учёт учеников, антропометрия, расчёты КСР, публичная страница прогресса по ссылке. Стек: **React**, **Vite**, **Tailwind CSS**, **Firebase** (авторизация и Firestore).

Папка **`legacy_code/`** — старый вариант на чистом HTML/JS (ориентир, не основной продукт). Рабочее приложение лежит в **`src/`**.

---

## Быстрый старт (для тебя или Антона)

1. Установи **Node.js 18+** (с сайта [nodejs.org](https://nodejs.org)).
2. В корне проекта скопируй файл с примером переменных:
   - Windows (PowerShell):  
     `Copy-Item .env.example .env.local`
3. Открой **`.env.local`** и заполни все строки `VITE_FIREBASE_*` значениями из **Firebase Console** → настройки проекта → твоё веб-приложение.
4. В терминале в папке проекта:
   - `npm install`
   - `npm run dev`
5. Открой в браузере адрес, который покажет Vite (обычно `http://127.0.0.1:5174`).

Без заполненного `.env.local` приложение не сможет подключиться к Firebase.

## Личный код ученика (шесть цифр)

- У каждого ученика в базе есть поле **`short_id`** — **ровно 6 цифр** (например `123456`). На экране он может отображаться с пробелом: **«123 456»** — для человека, в буфер обмена копируются **только цифры подряд**.
- Код нужен, чтобы **другой тренер** мог «подцепить» того же ученика: в приложении есть сценарий ввода **шести цифр** — вводишь код с карточки ученика у коллеги, система находит ученика и добавляет твоего тренера в список (много тренеров — через массив `coach_ids`).
- Если код не появился или не сохраняется, чаще всего дело в **правилах Firestore** или правах в консоли Firebase — это настраивает тот, кто подключал проект к базе.

Подробнее про интерфейс: карточка ученика → блок с **личным кодом** и кнопкой копирования; модальное окно добавления ученика — ввод **6 цифр** без пробелов.

## Публикация на Vercel (ссылка для Антона)

1. Залей проект на **GitHub** (см. раздел ниже).
2. Зайди на [vercel.com](https://vercel.com), войди через GitHub.
3. **Add New… → Project** → выбери этот репозиторий → **Import**.
4. Настройки сборки подхватятся из **`vercel.json`** (Vite, папка `dist`, маршруты SPA).
5. Обязательно: **Settings → Environment Variables** — добавь **те же имена**, что в `.env.example` (`VITE_FIREBASE_API_KEY`, `VITE_FIREBASE_AUTH_DOMAIN`, и т.д.), со значениями из Firebase. Сохрани и сделай **Redeploy**, если деплой уже был без переменных.
6. В **Firebase Console** → Authentication → **Authorized domains** добавь домен вида `твой-проект.vercel.app`, иначе вход с продакшн-URL может не сработать.

Готовая ссылка будет в Vercel на странице проекта (кнопка **Visit** / домен деплоя).

## GitHub и Cursor (пошагово, простыми словами)

### Связать папку на компьютере с GitHub

1. На [github.com](https://github.com) создай **новый репозиторий** (New repository), **без** галочек «добавить README / .gitignore» — пустой репозиторий, запомни адрес вида `https://github.com/ТВОЙ_ЛОГИН/ИМЯ_РЕПО.git`.
2. В **Cursor** открой эту папку проекта.
3. Слева открой значок **ветки / Git** (Source Control).
4. Если Git ещё не инициализирован: **Initialize Repository** (или в терминале один раз `git init`).
5. Внизу статус-бара или в Source Control найди **Publish Branch** / **Publish to GitHub** — выбери свой аккаунт и созданный репозиторий. Либо в терминале:
   - `git remote add origin https://github.com/ТВОЙ_ЛОГИН/ИМЯ_РЕПО.git`
   - `git branch -M main`
   - `git push -u origin main`

Важно: файл **`.env.local`** в репозиторий не попадёт (он в `.gitignore`) — так и должно быть, секреты не для GitHub.

### Как «запушить» изменения

1. Source Control → введи сообщение коммита (например: «Подготовка к деплою»).
2. Нажми **Commit**.
3. Нажми **Sync** / **Push** (или стрелка вверх «опубликовать»). После этого код появится на GitHub.

### Передать Антону

- Ссылку на **репозиторий** GitHub.
- Ссылку на **Vercel** (если уже задеплоил).
- Отдельно (в личку) пришли содержимое **`.env.local`** или попроси его завести свой Firebase — тогда дашь только доступ в консоль Firebase по роли.

## Legacy (`legacy_code/`)

Старые HTML-страницы подключают **`legacy_code/js/firebase-init.js`**. Там объект **`firebaseConfig`** нужно **заполнить вручную** теми же значениями, что и для `VITE_FIREBASE_*`, если хочешь кликать по legacy локально. Основной продукт для коллеги — **React-приложение** из `src/`.

---

## Правила Firestore

Файл **`firestore.rules`** в корне — пример правил для отладки. Их нужно **опубликовать** в Firebase Console (Firestore → Rules), если меняешь — иначе клиент получит ошибки доступа.
