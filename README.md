# Cartel Boxing App

Веб-приложение для тренера по боксу: список учеников, карточка с антропометрией, нормативами и техникой, данные в **Firebase** (Firestore + вход по почте). Есть публичная страница прогресса для родителей и спортсмена по ссылке «Поделиться прогрессом».

## Что нужно перед запуском

1. Установите [Node.js](https://nodejs.org/) (версия **18** или новее).
2. В корне проекта создайте файл **`.env.local`** (его **не** заливают в Git — он только у вас на компьютере).
3. Скопируйте содержимое из **`.env.example`** и заполните переменные значениями из [Firebase Console](https://console.firebase.google.com) → ваш проект → **Project settings** (шестерёнка) → раздел **Your apps** → веб-приложение.

Имена переменных должны начинаться с **`VITE_`**, как в примере — иначе Vite их не подставит в сборку.

## Команды

```bash
npm install
npm run dev
```

Сайт откроется на порту **5174** (см. `package.json`).

Сборка проверки:

```bash
npm run build
```

## Личный код ученика (6 цифр)

- У каждого ученика в базе есть **шестизначный код** (поле `short_id`): его видно в карточке ученика как **«Личный код»**.
- Тренер может **скопировать** код и передать другому тренеру. Второй тренер на главной нажимает **«Добавить ученика»** и вводит эти **6 цифр** — карточка того же ученика подключается к его аккаунту (если в **Firestore** разрешены нужные правила).
- Это **не** пароль от сайта и **не** ссылка для родителей. Для родителей — отдельная длинная ссылка **«Поделиться прогрессом»** (`/share/...`).

## Правила Firestore

В репозитории лежит файл **`firestore.rules`**. Его нужно **вручную скопировать** в Firebase Console → Firestore → **Rules** → **Publish**. Без этого вход и база могут не работать.

## Как залить проект на GitHub из Cursor

1. Зайдите на [github.com](https://github.com), войдите в аккаунт.
2. Нажмите **«+»** → **New repository**. Имя, например, `cartel-boxing-app`. Создайте **без** галочки «Add README» (чтобы не было лишнего первого коммита), или с README — тогда позже придётся подтянуть изменения — проще пустой репозиторий.
3. В **Cursor** откройте папку проекта.
4. Откройте панель **Source Control** (иконка ветки слева) или меню **View → SCM**.
5. Нажмите **«Publish to GitHub»** / **«Опубликовать на GitHub»** (если Cursor предложит связать аккаунт GitHub — войдите и разрешите доступ).
6. Выберите репозиторий (созданный на шаге 2) и подтвердите публикацию. Cursor сделает первый **push** сам.

Если кнопки «Publish to GitHub» нет: в терминале Cursor один раз выполните (подставьте свой URL репозитория):

```bash
git init
git add .
git commit -m "Первый коммит: Cartel Boxing App"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/cartel-boxing-app.git
git push -u origin main
```

**Важно:** файл **`.env.local`** с ключами в Git не попадёт (он в `.gitignore`). Антону нужно будет создать свой `.env.local` по **`.env.example`**.

## Как «запушить» изменения позже

1. В **Source Control** посмотрите список изменённых файлов.
2. Введите короткое сообщение коммита (например: «Исправлена форма логина»).
3. Нажмите **Commit**.
4. Нажмите **Sync** / **Push** (или стрелка «вверх» у ветки) — код уйдёт на GitHub.

## Как задеплоить на Vercel «в два клика»

1. Зайдите на [vercel.com](https://vercel.com), войдите (можно через GitHub).
2. **Add New… → Project** → выберите репозиторий **cartel-boxing-app** (или как вы назвали) → **Import**.
3. Настройки сборки Vercel обычно подхватывает сам: **Build Command** `npm run build`, папка вывода **`dist`**. В корне есть **`vercel.json`** — маршруты вроде `/share/...` отдадутся приложению, а не «страница не найдена».
4. Раскройте **Environment Variables** и **один в один** добавьте все переменные из **`.env.example`** (те же имена `VITE_FIREBASE_...`), вставьте значения из Firebase.
5. Нажмите **Deploy**. Через минуту появится ссылка вида `https://что-то.vercel.app` — её можно отправить Антону.

После смены переменных в Firebase или в коде: снова **Redeploy** в Vercel или просто новый **push** в GitHub — Vercel часто пересобирает сам.

**Важно для ссылки Vercel:** в Firebase Console откройте **Authentication** → **Settings** → **Authorized domains** и добавьте домен вида `ваш-проект.vercel.app` (иначе вход по почте на продакшене может не сработать).
